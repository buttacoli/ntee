ntee by buttacoli

A linux tee-like emulator for processes communicating through network socket IPC. 

Usage: ntee [-h|--help] [-o <path>] [--sock <tcp|udp>] [-p <N>]
             -L <host> <port> -R <cmd> [@NTEEPORT] [args...]

Purpose: NTEE is a program which sits between two other programs communicating
         through sockets.  As traffic comes between programs L and R, ntee
         records the timing and data being sent for further analysis or play-
         back.  The options in the command line usage reflect that there are
         two programs, a left side process 'L', and right side process 'R'
         with ntee sitting in the middle.

   +---+                           +------+                         +---+
   | L |---listen---><---connect---| ntee |----fork/exec----------->| R |
   |___|                           |______|---listen--><--connect---|___|

Flags:
  -o <path>         Path to where the output should be written.
                    Defaults to ./ntee_output
  --sock <tcp|udp>  Defines the socket type which the L and R side processes
                    are using.  Defaults to tcp.
  -p <int>          Forces the port number upon which ntee will serve to the
                    R side process.  This is here in case the R program has
                    its connection hardcoded to use a certain port.
  -L <ip> <int>     The ip address and port number of the L side process.
                    ntee will connect to this process after the R side program
                    has been started and decides to connect with ntees service
                    port.
  -R <cmd> [args]   The user specifies the R side process by giving its
                    command line argument, the special argument @NTEEPORT can
                    be inserted anywhere in the args where the port number
                    that the process would connect would ordinarily be specified
                    with.  Ntee will also define the $NTEEPORT environment var
                    to the value which is being used for the ntee service port.

Notes:
  Both the -L and -R arguments must be specified for ntee to start properly.

Dependencies

  • gcc-c++-4.4.3-4.fc12.i686
  • doxygen-1.6.1-1.fc12.i686
  • boost-1.39.0-9.fc12.i686
  • binutils-2.19.51.0.14-37.fc12.i686

Install

  • git clone
  • cd to the src/ directory
  • type: make all
